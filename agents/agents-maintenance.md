# Agent 规则维护规范

## 适用范围

本规范适用于根目录 `AGENTS.md`、`agents/` 下的规则文件，以及未来新增的嵌套 `AGENTS.md`。

## 修改要求

- 根目录 `AGENTS.md` 仅维护入口、索引、适用范围和读取顺序，不堆叠具体规则。
- 具体规则按主题存放在 `agents/` 中，一个文件只负责一个明确主题。
- 新增规则文件时，必须同步更新根目录 `AGENTS.md` 的索引和读取条件。
- 删除或重命名规则文件时，必须同步修复所有引用链接。
- 规则使用明确、可执行、可验证的表述，避免“适当处理”等模糊要求。
- 避免在多个文件重复同一规则；应链接或指定唯一权威文件。
- 规则必须与仓库当前技术栈和目录结构一致，不保留失效说明。
- 不在规则中写入密钥、个人路径、个人偏好或仅适用于单次任务的临时要求。

## 嵌套规则

- 仅当子目录确实需要不同规则时创建嵌套 `AGENTS.md`。
- 嵌套规则应说明适用范围，只补充或覆盖必要部分。
- 修改嵌套规则时检查其与上级规则是否冲突；有意覆盖时必须明确说明。

## 验证清单

- 检查 Markdown 链接均指向存在的文件。
- 检查新增规则是否放在职责正确的文件中。
- 检查是否存在重复、冲突、失效或不可执行的规则。
- 检查 `git diff`，确保未包含无关修改。
