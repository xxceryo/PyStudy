# 测试规范

- 后端使用 `pytest` 编写和运行自动化测试。
- 前端当前未配置测试框架；新增可测试行为时，应先配置适合 Vue 和 TypeScript 的测试工具并同步更新本规范。
- 修复缺陷时添加能够复现缺陷的回归测试。
- 新增行为时覆盖主要成功路径，并根据风险覆盖错误和边界路径。
- 测试应独立、可重复，不依赖执行顺序或外部共享状态。
- 避免真实访问生产数据库、Redis 或外部服务；使用隔离环境、替身或依赖覆盖。
- 测试名称应清楚描述被测行为和预期结果。
- 优先运行与改动相关的测试，完成前运行完整测试集：

```powershell
uv run pytest
Set-Location frontend
npm run build
```

- 前端改动完成前至少运行 `npm run build`，验证 TypeScript 类型检查和 Vite 构建。
- 未能运行测试或构建时，必须明确说明原因和未验证范围。
